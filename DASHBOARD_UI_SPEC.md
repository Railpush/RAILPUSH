### 4.2 Breadcrumb Navigation

Located at the top of the main content area, above the page title:

```
Workspace Name  /  Project Name  /  production  /  my-api
```

- Each segment is clickable
- Current page is not linked (plain text)
- Uses `>` or `/` as separators
- Font: `text-body-sm`, color: `--text-secondary`

### 4.3 Global Search (Command Palette)

- Triggered by `Cmd+K` (Mac) or `Ctrl+K` (Windows/Linux)
- Centered modal overlay with search input
- Results grouped by type: Services, Databases, Blueprints, Settings
- Keyboard navigation with arrow keys
- Shows shortcuts to common settings (API keys, webhooks)

**Log Entry Design**:
- Monospace font (`JetBrains Mono` or `SF Mono`)
- Dark background (even darker than dashboard in dark mode): `#0D0D14`
- Line components:
  - **Level icon**: Hidden for `info`, visible on hover; icons for warn (yellow triangle), error (red circle), debug (gray)
  - **Timestamp**: `text-xs`, `--text-tertiary`, shows local time; hover reveals UTC + Unix timestamp
  - **Instance ID**: `text-xs`, clickable to filter, shown in brackets `[i-abc123]`
  - **Message**: `text-body-sm`, white/light text
- HTTP log lines show: `METHOD /path STATUS_CODE DURATION`
- Color coding per level:
  - `info`: default text color
  - `warning`: `#FFBB33` (yellow)
  - `error`: `#FF4D6A` (red)
  - `debug`: `--text-tertiary` (dim)

**Controls**:
- **Live tail toggle**: Green dot when active, streams new entries
- **Search**: Text input with regex support (RE2 syntax)
- **Named filters**: `level:`, `instance:`, `method:`, `status_code:`, `host:`, `path:`
- **Time range selector**: Preset options (Last 1h, 6h, 24h, 7d) + custom range
- **Keyboard shortcuts**: `/` = focus search, `M` = fullscreen, `Home`/`End` = jump

## 12. Key UX Patterns

### 12.1 Context-Aware Navigation
The sidebar always shows what is most relevant to the current view. When looking at the workspace, it shows all resource types. When viewing a specific service, it shows that service's sub-pages (Logs, Metrics, Settings, etc.).

### 12.2 Three-Option Save for Environment Variables
After editing env vars, a split button offers three deployment strategies:
1. Save, rebuild, and deploy
2. Save and deploy (skip rebuild)
3. Save only (no deploy)

### 12.3 Deploy Timeline
Events page shows a chronological timeline of all deployment events, each clickable to view build logs and deploy details.

### 12.4 Copy-to-Clipboard
All connection strings, URLs, and credentials have a clipboard copy button (clipboard icon) that shows a brief "Copied!" confirmation.

### 12.5 Secret Masking
Secret values displayed as dots (`в—Ҹв—Ҹв—Ҹв—Ҹв—Ҹв—Ҹв—Ҹв—Ҹ`) with a "Show" toggle button and "Edit" button. Values are never exposed in URL parameters or logs.

### 12.6 Keyboard Shortcuts
- `Cmd+K` / `Ctrl+K`: Global search
- `/`: Focus log search
- `M`: Fullscreen logs
- `Home`: Jump to log start
- `End`: Jump to log end

### 12.7 Empty States
Empty states include:
- Illustration or icon
- Descriptive text explaining what should be here
- Primary action button (e.g., "Create your first service")

### 12.8 Confirmation Dialogs
Destructive actions (delete service, delete database) require:
- Modal with warning text
- User must type the service/resource name to confirm
- Red "Delete" button only enabled after name match
